# **App Name**: Konki Burger Web App

## Core Features:

- User Authentication (Optional): Allow users to register and log in, or proceed as a guest.
- Product Browsing: Display products with filtering by categories.
- Shopping Cart: Implement a shopping cart functionality (add, remove, modify quantities).
- Order Checkout: Process the order and send the data to the backend API.
- Admin Product Management: CRUD operations for managing products and categories. New products should automatically appear in the customer interface.
- Admin User Management: List, delete, and modify registered users (email, address, password reset).
- Order Flow Management: Real-time table of pending orders with actions to accept (requiring pickup time) or reject. Simulate sending an email notification with the assigned time (or rejection notice).

## Style Guidelines:

- Primary color: A vibrant red (#E53935), symbolizing energy and the appetizing nature of burgers. The choice of red is based on the desire to capture the spirit of the American diner.
- Background color: black(#191919) creates a welcoming and bright background that keeps the emphasis on the food images.
- Accent color: Yellow (#FFEB3B) used sparingly for interactive elements and call-to-action buttons.
- Body font: 'PT Sans', a sans-serif font offering modern and warmth aesthetics to maintain the appâ€™s contemporary feel.
- Headline font: 'Space Grotesk', a sans-serif font providing the web app with a computerized and scientific vibe to enhance readability and create visual contrast.
- Use simple, outline-style icons to represent categories and actions throughout the app.
- Subtle transitions and animations on button hover states and when adding items to the cart to enhance user experience.
